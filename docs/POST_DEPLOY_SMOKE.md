# בדיקות אחרי כל Deploy - NeverMind

הרץ את הרשימה הזו אחרי כל פריסה לפרודקשן (או ל-staging לפני מעבר). מטרה: לתפוס רגרסיות בסביבה, בנכסים סטטיים ובשירותים חיצוניים.

## 1. חיפוש (Pagefind)

- [ ] פתח את האתר ולחץ על כפתור החיפוש בכותרת.
- [ ] הרץ שאילתה שידוע שיש לה תוצאות (מילה ממאמר קיים).
- [ ] ודא שאין שגיאת רשת ל-`/pagefind/pagefind.js` או לקבצי אינדקס (ב-DevTools → Network).

## 2. מאמר פרימיום

- [ ] פתח מאמר עם `isPremium: true`.
- [ ] בלי סשן מועדון: ודא שהמקטע אחרי `[[premium-break]]` או המנגנון הרלוונטי מוסתר/מוגבל כמצופה.
- [ ] עם סשן מועדון תקין: ודא גישה מלאה.

## 3. טפסים (Web3Forms)

- [ ] שלח טופס יצירת קשר / דוגמה ממוסגר באתר.
- [ ] ודא הגעה ליעד (מייל או לוח Web3Forms) או תשובת הצלחה ב-UI.

## 4. מועדון (Webhook)

- [ ] טופס כניסת מועדון עם פרטים בדויים - ודא תגובת הצלחה או הודעת שגיאה ברורה (לא 500 שקט).
- [ ] אם יש Worker: בדוק לוגים ב-Cloudflare Workers לפי הצורך.

## 5. קישורים חיצוניים (דגימה)

- [ ] קישור וואטסאפ מהכותרת/תחתית.
- [ ] קישורי רשתות מהכותרת (אינסטגרם, טיקטוק וכו') - דגימה אחת מספיקה לפריסה שגרתית.
- [ ] כל קישור ששינית בקונפיג באותה גרסה.

## 6. ניהול (Zero Trust)

- [ ] ממכשיר לא מחובר למדיניות: `/dashboard/` דורש התחברות Access.
- [ ] ממייל מורשה: כניסה תקינה.

## 7. אנליטיקס (אם מופעל)

- [ ] ודא שאין שגיאות טעינה בקונסולה מסקריפט מעקב.

---
